# exitTime
Изначальный плагин: https://www.blast.hk/threads/61048/
Идея навеяна этим плагином: https://www.blast.hk/threads/51215/
Дело в том, что у сампа есть свой таймер выхода из игры, который стартует при деинициализации, после сохранения настроек, но перед выгрузкой объектов и прочего. По дефолту этот таймер настроен на 2 секунды (2000мс), а данный плагин позволяет изменить это время.

Для поддерживаемых сборок `samp.dll` тем же значением из INI дополнительно патчится задержка **500 ms** в общей цепочке выхода по чат-команде (**/quit**, **/q** — один и тот же `push imm32` в типичной сборке 0.3.7) — RVA imm32 в `kSupportedVersions` (`quitDelayImm32Rva`). Таблица по entry point: **R1, R2, R3, R3-1, R4, R4-2, R5-1, DL-R1**. Сборки **R5-2** и **DL-R1-2** используют те же entry point, что **R5-1** и **DL-R1**, поэтому попадают под те же строки без отдельных записей.

Дополнительно для **gta_sa.exe** (GTA San Andreas **1.0 US** — compact или Hoodlum, та же проверка dword на **0x401000**, что и в plugin-sdk `GameVersion.cpp`): во время `CGame::Shutdown` (RVA **0x53C900**) перехватывается `kernel32!Sleep`: при значении **> 0** каждый вызов `Sleep` в этом окне ограничивается сверху указанным числом миллисекунд; при **0** запросы заменяются на `Sleep(0)`; **-1** / **-2** — немедленный `ExitProcess` / `TerminateProcess` до штатного shutdown. На других сборках exe эта часть не активируется (смещения не подставляются).

**Настройки**

Настройка таймера осуществляется в файле exitTime.ini. Так как шаг таймера в samp имеет 100мс, то разницы между 105, 150, 199 и 200мс не будет, а вот разница между 100 и 200мс будет.

Если указать **0мс**, то дополнительно в игре отключатся все вызовы Sleep, что потенциально может дополнительно сократить время выхода.

Если указать **-1мс**, при срабатывании хука (SA:MP — первый вызов перехваченного `GetTickCount` при выгрузке, если для этой сборки задано смещение вызова; **R2** в статическом импорте не тянет `GetTickCount` — для неё только GTA-часть; GTA — вход в `CGame::Shutdown`) вызывается **ExitProcess**.

Если указать **-2мс**, в тех же точках вызывается **TerminateProcess** для текущего процесса.

Пересобрано и переделано нейросетью под перечисленные версии `samp.dll`.

Файл **exitTime.ini** читается и создаётся/обновляется в каталоге загруженного **exitTime.asi** (тот же путь, что даёт `GetModuleFileName` для модуля плагина; удобно с modloader). По умолчанию в ключе **1000** мс.

Можно поместить \modloader\asi\exitTime.asi или папку scripts
